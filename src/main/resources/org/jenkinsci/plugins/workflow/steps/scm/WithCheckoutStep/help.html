This step allows checking out any Pipeline-compatible SCM, with any environment variables set by that SCM made available within the block. To use a concrete SCM implementation, just install the corresponding plugin and check if it is shown in the list below. Then select the SCM to use from the dropdown list and configure it as needed.

For example:

        node {
            withCheckout(scm) {
                echo "GIT_COMMIT is ${env.GIT_COMMIT}"
            }
        }

Note that if multiple withCheckout invocations are nested within each other, only the closest parent invocation's environment variables will be available within that body.
